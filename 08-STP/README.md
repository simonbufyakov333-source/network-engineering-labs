# Лабораторная 08 — STP/RSTP

## Цель

Создать резервную L2-топологию без петель и управлять выбором root bridge и заблокированного порта.

## Вводные

Три коммутатора образуют треугольник: `SW-CORE`, `SW-ACC-01`, `SW-ACC-02`. Между ними работают VLAN 10, 20 и 30.

## Требования

- Использовать Rapid PVST+.
- Назначить `SW-CORE` root primary.
- Назначить `SW-ACC-01` root secondary.
- Проверить root/designated/alternate роли портов.
- На пользовательских портах включить PortFast и BPDU Guard.
- Не включать PortFast на межкоммутаторных trunk.
- Изменить стоимость одного линка и зафиксировать смену выбранного пути.

## Проверки

- В топологии отсутствует broadcast storm.
- Root bridge выбран согласно проекту.
- Один из резервных путей находится в alternate/discarding.
- При отключении рабочего trunk резервный переходит в forwarding.
- BPDU Guard отключает пользовательский порт при подключении коммутатора.

## Неисправности

1. Root bridge стал access-коммутатором.
2. PortFast включён на trunk.
3. Несовпадающий STP mode.
4. Непредвиденный путь выбран из-за port cost.
5. Err-disabled порт не восстановлен после BPDU Guard.

## Результаты

Сохрани схему с ролями портов, конфигурации, `show spanning-tree` до/после отказа и измеренное время восстановления связи.
