# Лабораторная 04 — OSPF

## Цель

Построить многозонную OSPF-сеть и научиться проверять соседства, выбор маршрутов и причины отсутствия анонсов.

## Вводные

Четыре маршрутизатора: `R-HQ-01`, `R-HQ-02`, `R-BR-01`, `R-BR-02`. Центральные соединения относятся к area 0, филиал 1 — area 10, филиал 2 — area 20. Адреса использовать из `10.40.0.0/16`. На каждом маршрутизаторе создать loopback /32.

## Требования

- OSPF process 10 и явный router-id.
- Area 0 должна быть непрерывной.
- Пользовательские интерфейсы сделать passive.
- Анонсировать только необходимые сети.
- Настроить стоимость одного канала так, чтобы был виден выбор лучшего пути.
- Добавить default route на HQ и распространить её через OSPF.
- По возможности включить OSPF-аутентификацию.

## Проверки

- Все ожидаемые соседства находятся в FULL.
- Loopback всех маршрутизаторов доступны.
- Пользовательские интерфейсы не формируют соседств.
- При отключении основного канала выбирается альтернативный путь.
- После восстановления таблица маршрутизации сходится автоматически.

## Неисправности

1. Несовпадение area.
2. Несовпадение hello/dead timers.
3. Ошибка wildcard mask.
4. Дублирующийся router-id.
5. Неверная аутентификация.
6. Default route не создана до команды `default-information originate`.

## Результаты

Приложи схему с areas, адресный план, конфигурации, `show ip ospf neighbor`, `show ip ospf interface brief`, `show ip route ospf` и диагностику каждого сбоя.
