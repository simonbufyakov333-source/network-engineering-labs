# Лабораторная 03 — Статическая маршрутизация

## Цель

Настроить связность центрального офиса и двух филиалов, используя статические, default и floating static routes.

## Вводные

Маршрутизаторы: `R-HQ`, `R-BR1`, `R-BR2`. У каждого есть локальная сеть. Между HQ и BR1 предусмотрены основной и резервный каналы. Внутренние адреса выделить из `10.30.0.0/16`, транзитные сети использовать с маской /30 или /31, если образ поддерживает.

## Требования

- Подготовить таблицу адресации.
- Настроить точные статические маршруты.
- В филиалах использовать default route в сторону HQ.
- Для BR1 создать floating static route с большей administrative distance.
- Исключить маршрутные петли.
- Добавить описания интерфейсов.

## Проверки

- Все LAN доступны друг другу.
- `traceroute` проходит через ожидаемые маршрутизаторы.
- При отказе основного канала BR1 использует резервный.
- После восстановления возвращается основной маршрут.
- Неизвестные сети в филиале направляются только в HQ.

## Неисправности

1. Неверный next-hop.
2. Маска маршрута шире необходимой.
3. Резервный маршрут имеет меньшую administrative distance.
4. На обратной стороне отсутствует маршрут.
5. Интерфейс up, но next-hop недоступен на L2.

## Результаты

Сохрани схему, таблицу адресации, конфигурации, routing table и traceroute до/после отказа. В отчёте отдельно объясни рекурсивный lookup и выбор longest prefix match.
