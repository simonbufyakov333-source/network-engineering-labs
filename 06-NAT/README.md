# Лабораторная 06 — NAT и PAT

## Цель

Организовать выход внутренних сетей в интернет через PAT и публикацию тестового сервера через static NAT.

## Вводные

Маршрутизаторы `R-EDGE` и `ISP`, внутренние сети USERS и SERVERS, внешний тестовый сервер. Внутренние адреса — RFC1918, внешние тестовые адреса — только из диапазонов RFC 5737.

## Требования

- Пометить интерфейсы как NAT inside/outside.
- Настроить PAT для пользовательских подсетей через внешний интерфейс.
- Настроить static NAT для внутреннего web-сервера.
- Ограничить NAT ACL только нужными внутренними сетями.
- Создать default route к ISP.
- Не использовать реальные публичные адреса домашней сети.

## Проверки

- Несколько клиентов одновременно выходят наружу через один адрес.
- В таблице трансляций различаются порты.
- Web-сервер доступен по статическому внешнему адресу.
- Неразрешённая внутренняя сеть не транслируется.
- После очистки translations новые записи создаются повторно.

## Неисправности

1. Inside/outside перепутаны.
2. NAT ACL не совпадает с адресами клиентов.
3. Нет default route.
4. Static NAT использует занятый адрес.
5. Обратный маршрут на ISP отсутствует.

## Результаты

Сохрани конфигурации, `show ip nat translations`, `show ip nat statistics`, routing table и результаты одновременных подключений двух клиентов.
