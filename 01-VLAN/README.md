# Лабораторная 01 — VLAN и trunk

## Цель

Разделить офисную сеть на логические сегменты и передать несколько VLAN между коммутаторами по 802.1Q trunk.

## Вводные

Оборудование: два Cisco 2960 (`SW-01`, `SW-02`) и минимум четыре ПК. Отделы расположены на обоих коммутаторах.

| VLAN | Имя | Назначение |
|---:|---|---|
| 10 | USERS | Пользователи |
| 20 | IT | ИТ-отдел |
| 30 | SERVERS | Серверы |
| 99 | MGMT | Управление |
| 999 | NATIVE-BLACKHOLE | Неиспользуемый native VLAN |

## Требования

- Создать VLAN на обоих коммутаторах.
- Назначить access-порты согласно самостоятельно составленной таблице.
- Настроить 802.1Q trunk и разрешить только VLAN 10, 20, 30 и 99.
- Использовать VLAN 999 как native VLAN.
- Отключить DTP там, где это поддерживается.
- Свободные порты перевести в VLAN 999 и выключить.
- Не настраивать межвлановую маршрутизацию.

## Проверки

- Узлы одного VLAN на разных коммутаторах доступны.
- Узлы разных VLAN недоступны.
- В trunk передаются только разрешённые VLAN.
- Native VLAN совпадает на обеих сторонах.

## Неисправности для диагностики

1. Один access-порт помещён в неправильный VLAN.
2. VLAN 20 отсутствует в allowed list на одном конце trunk.
3. Native VLAN не совпадает.
4. Порт с ПК ошибочно настроен как trunk.

## Что сохранить

Добавь схему, таблицу портов, файл Packet Tracer, конфигурации `SW-01.cfg` и `SW-02.cfg`, а также вывод `show vlan brief` и `show interfaces trunk`. В отчёте описывай: симптом → команды → причина → исправление → повторная проверка.
