# Лабораторная 05 — DHCP и DHCP Relay

## Цель

Централизованно выдавать адреса нескольким VLAN и диагностировать полный процесс DORA.

## Вводные

VLAN 10 USERS, VLAN 20 IT и VLAN 30 GUEST находятся за L3-коммутатором. DHCP-сервер расположен в VLAN 50 SERVERS. Подсети выбрать из `10.50.0.0/16`.

## Требования

- Создать отдельный DHCP pool для каждого клиентского VLAN.
- Исключить адреса шлюзов, серверов и сетевого оборудования.
- Выдавать gateway и DNS.
- Настроить DHCP relay (`ip helper-address`) на правильных SVI.
- Назначить одному устройству статический адрес вне динамического диапазона.
- Проверить срок аренды и повторное получение адреса.

## Проверки

- Клиенты каждого VLAN получают адрес своей подсети.
- Gateway и DNS передаются корректно.
- Сервер видит запросы, пришедшие через relay.
- После освобождения и обновления аренды клиент сохраняет связность.
- Клиент из неизвестного VLAN не получает адрес.

## Неисправности

1. `ip helper-address` установлен на серверном SVI вместо клиентского.
2. В pool указана неправильная network mask.
3. Gateway попал в динамический диапазон.
4. Между relay и сервером отсутствует обратный маршрут.
5. ACL блокирует UDP 67/68.

## Результаты

Добавь схему, конфигурации, таблицу DHCP bindings и кратко опиши DORA: Discover, Offer, Request, Acknowledge.
