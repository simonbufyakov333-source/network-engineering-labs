# Лабораторная 02 — Inter-VLAN Routing

## Цель

Обеспечить контролируемую маршрутизацию между VLAN методом Router-on-a-Stick, а затем повторить решение на L3-коммутаторе.

## Вводные

Используй VLAN 10 USERS, VLAN 20 IT, VLAN 30 SERVERS и VLAN 99 MGMT. Адресный блок — `10.20.0.0/16`; подсети /24 выбери самостоятельно. Оборудование: маршрутизатор `R-01`, коммутатор `SW-01`, три клиентских узла.

## Часть A — Router-on-a-Stick

- Создать subinterface для каждого VLAN.
- Настроить 802.1Q encapsulation и адреса шлюзов.
- Между `SW-01` и `R-01` настроить trunk.
- Для native VLAN не использовать пользовательскую сеть.
- На клиентах указать правильные gateway.

## Часть B — L3 switch

Повторить маршрутизацию с использованием SVI и `ip routing`. Сравнить оба варианта по отказоустойчивости, производительности и применению.

## Проверки

- Клиенты достигают своих шлюзов.
- Между разрешёнными VLAN проходит ICMP.
- После выключения `ip routing` на L3-коммутаторе межвлановая связь пропадает.
- В routing table присутствуют connected-сети.

## Неисправности

1. Неверный VLAN ID на subinterface.
2. Физический интерфейс маршрутизатора shutdown.
3. На клиенте указан неправильный gateway.
4. SVI down из-за отсутствия активных портов VLAN.

## Результаты

Сохрани обе топологии, адресный план, конфигурации и вывод `show ip interface brief`, `show ip route`, `show interfaces trunk`. Объясни, какой вариант выбрал бы для офиса на 50 человек.
