# Лабораторная 10 — Безопасное администрирование по SSH

## Цель

Настроить безопасный удалённый доступ к сетевому оборудованию и ограничить его административной сетью.

## Вводные

Маршрутизатор `R-01`, коммутатор `SW-01`, пользовательская сеть и VLAN 99 MGMT. Администратор работает только из IT/MGMT.

## Требования

- Настроить hostname, domain name и RSA-ключи.
- Использовать SSH version 2.
- Создать локального пользователя с privilege и secret.
- Включить `login local` на VTY.
- Запретить Telnet.
- Ограничить VTY standard ACL только сетью IT/MGMT.
- Настроить exec-timeout и login blocking.
- Добавить предупреждающий banner.
- Пароли и ключи не публиковать.

## Проверки

- SSH из административной сети работает.
- SSH из пользовательской сети запрещён.
- Telnet недоступен.
- Сессия завершается по timeout.
- В конфигурации нет паролей в открытом виде.

## Неисправности

1. Не задан domain name, RSA-ключи не создаются.
2. На VTY отсутствует `login local`.
3. ACL разрешает неправильную сеть.
4. Разрешён `transport input all`.
5. Устройство недоступно из-за ACL — восстановить через console.

## Результаты

Добавь обезличенные конфигурации, результаты успешного и отклонённого входа и объяснение разницы между enable secret, local user и AAA.
