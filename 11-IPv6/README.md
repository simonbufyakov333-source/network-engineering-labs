# Лабораторная 11 — IPv6

## Цель

Настроить IPv6-адресацию, SLAAC/DHCPv6 и маршрутизацию между несколькими сетями.

## Вводные

Два маршрутизатора и три LAN. Использовать документационный префикс `2001:db8::/32`. Для каждой LAN выделить /64. Создать link-local адреса и loopback.

## Требования

- Включить IPv6 unicast routing.
- Назначить global unicast и link-local адреса.
- Проверить SLAAC.
- Настроить stateless либо stateful DHCPv6 и объяснить выбор.
- Настроить статический IPv6-маршрут, включая default.
- Повторить динамическую маршрутизацию с OSPFv3.
- Не дробить пользовательскую LAN меньше /64.

## Проверки

- Клиенты получают ожидаемые IPv6-адреса.
- Neighbor Discovery формирует соседей.
- Все LAN доступны.
- Default route используется для внешней тестовой сети.
- OSPFv3-соседство восстанавливается после отказа линка.

## Неисправности

1. Не включён `ipv6 unicast-routing`.
2. Ошибочный /64.
3. Default route указывает неправильный link-local next-hop/interface.
4. Router Advertisement отключён.
5. OSPFv3 включён не на том интерфейсе.

## Результаты

Сохрани адресный план, конфигурации, `show ipv6 interface brief`, `show ipv6 neighbors`, `show ipv6 route` и сравнение ARP с NDP.
