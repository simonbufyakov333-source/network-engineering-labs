# Лабораторная 09 — EtherChannel и LACP

## Цель

Объединить физические линии в логический канал, увеличить пропускную способность и сохранить связь при отказе одного участника.

## Вводные

Коммутаторы `SW-CORE` и `SW-ACC` соединены четырьмя GigabitEthernet-портами. По каналу передаются VLAN 10, 20, 30 и 99.

## Требования

- Собрать LACP Port-Channel минимум из двух линий.
- Использовать режимы active/passive так, чтобы агрегация формировалась.
- Настраивать trunk-параметры согласованно.
- Проверить load-balancing method.
- Сравнить LACP с PAgP и статическим mode on.
- Убедиться, что STP видит Port-Channel как один интерфейс.

## Проверки

- Все участники находятся в bundled state.
- VLAN проходят через Port-Channel.
- Отключение одного физического интерфейса не прерывает связь.
- Пропускная способность логического интерфейса отражает агрегацию.
- После возврата порт автоматически включается в группу.

## Неисправности

1. Разные channel-group.
2. Несовпадающие allowed VLAN.
3. Разная speed/duplex.
4. Обе стороны passive.
5. Один порт настроен как access.

## Результаты

Приложи конфигурации, `show etherchannel summary`, `show interfaces port-channel`, STP-состояние и диагностику минимум трёх ошибок.
