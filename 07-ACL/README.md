# Лабораторная 07 — ACL и сегментация

## Цель

Реализовать матрицу доступа между отделами и научиться выбирать правильное место и направление применения ACL.

## Вводные

Сегменты: USERS, IT, SERVERS, GUEST и MGMT. Межвлановая маршрутизация уже работает.

## Матрица доступа

| Источник | Назначение | Разрешить |
|---|---|---|
| USERS | SERVERS | DNS и HTTPS |
| IT | MGMT | SSH, SNMP, ICMP |
| GUEST | Внутренние сети | Запретить |
| GUEST | Интернет | DNS, HTTP, HTTPS |
| Остальной трафик | — | По явно описанной политике |

## Требования

- Использовать именованные extended ACL.
- Размещать ACL близко к источнику, если это не ломает другие потоки.
- Добавить комментарии/remark.
- Применять правила в корректном направлении.
- Учесть implicit deny.
- Не блокировать административный доступ до проверки альтернативного подключения.

## Проверки

Для каждой строки матрицы выполнить позитивный и негативный тест. Проверить счётчики ACE до и после трафика.

## Неисправности

1. Общий deny расположен выше permit.
2. Source и destination перепутаны.
3. Wildcard mask задана неверно.
4. ACL применён outbound вместо inbound.
5. Разрешён TCP-порт сервера, но DNS использует UDP.

## Результаты

Добавь схему, матрицу с фактическими IP, конфигурации, `show access-lists` и таблицу тестов: источник, назначение, протокол, ожидаемый и фактический результат.
